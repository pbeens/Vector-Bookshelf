This workflow guides you through updating the project documentation files.

## Steps

1. **Review Recent Changes**
   - Check git history or recent development activity
   - Identify new features, bug fixes, or architectural changes
   - Note any breaking changes or migration requirements

2. **Update DEVELOPMENT_LOG.md**
   - Add a new session entry under "Development Iterations"
   - Include date, objective, key changes, and technical details
   - Document both frontend and backend modifications
   - Explain the reasoning behind major decisions

3. **Update README.md**
   - Verify project description is accurate
   - Update tech stack if new dependencies were added
   - Revise "How It Works" section if workflow changed
   - Update setup instructions if requirements changed
   - Add documentation for new features or commands

4. **Verify Consistency**
   - Ensure version numbers match (if applicable)
   - Check that feature descriptions align across documents
   - Validate all code examples work
   - Confirm file links are correct
   - Review markdown formatting

5. **Review and Commit**
   - Read through both updated files
   - Ensure clarity and completeness
   - Commit changes with descriptive message

## Notes

- Document while details are fresh in your mind
- Be specific about what changed and why
- Use clear headers and formatting for scannability
- Include links to relevant code files when helpful
